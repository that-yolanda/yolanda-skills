# 页面模板

Entity 和 Topic 的内容生成规范。本文件是提取维度，输出格式和写作规则的唯一定义处。

## 封面选图

根据主题领域自动选择封面图，无需询问用户，后续允许用户自己修改。

### 获取可用图片

```bash
ls "{cover_dir}"/*.jpg 2>/dev/null | xargs -n1 basename
```

`{cover_dir}` 从 EXTEND.md 读取。根据返回的文件名列表，选择与页面主题最匹配的图片（按文件名语义判断，如 `bot.jpg` → 人工智能，`terminal.jpg` → 代码，`book.jpg` → 阅读）。若无法匹配，选最通用的图片。

## Entity

具体知识节点，聚焦一个特定事物，技术或概念。

提取维度：
- **一句话定义**：100字以内，说清它是什么
- **核心特征**：3-5个要点，每个要点 = 判断句 + 一句补充（为什么成立/什么场景适用/怎么操作），2-3句封顶
- **关联**：与知识库中哪些其他页面有直接联系
- **增量与冲突**：对已有知识补充了什么，是否与旧知识矛盾。判断三个方向：
  - 新观点：是否提出了不同于已有认知的见解
  - 新案例：是否提供了新的实证或应用场景
  - 新方法论：是否引入了新的方法、工具或框架

Frontmatter：

```yaml
---
tags: Entity
description: "{100字以内定义}"
cover: "[[{image_name}.jpg]]"
---
```

正文：

```markdown
## 核心理解

> 100字以内定义

## 核心特征

- {判断句 + 补充句，每条2-3句} [[源文件名#章节名]]
- {判断句 + 补充句，每条2-3句} [[源文件名#章节名]]

## 更新日志

- [t] YYYY-MM-DD：{增量描述 / 与旧知识矛盾} — [[源文件名]]

## 关联

- [>] [[页面名]] — 关系描述
```

## Topic

宏观综合页面，聚合多个知识点的主题。

提取维度：
- **核心论点**：该主题目前的宏观结论
- **支撑框架**：构成该主题的关键 Entity 及各自角色
- **演进与挑战**：新来源加强了什么观点，提出了什么新挑战

Frontmatter：

```yaml
---
tags: Topic
description: "{100字以内定义}"
cover: "[[{image_name}.jpg]]"
---
```

正文：

```markdown
## 核心理解

> 该主题的宏观结论

## 支撑框架

- [[Entity名]] — 在主题中的角色
- [[Entity名]] — 在主题中的角色

## 更新日志

- [t] YYYY-MM-DD：{新来源加强了什么 / 提出了什么挑战} — [[源文件名]]

## 关联

- [>] [[页面名]] — 关系描述
```

## 规则

- `description` 不超过 100 字，与 `## 核心理解` 保持一致
- 用自己的语言浓缩，不搬运原文，不嵌入大段引用或代码块
- `## 核心特征` 每条 = 判断句 + 一句补充（为什么/怎么做/适用场景），2-3句封顶，禁止纯概念名和大段引用
- `## 核心特征` 每条末尾可附 `[[源文件名#章节名]]`，精确指向来源章节，非必填
- `## 更新日志` 每次更新追加一条，必须说明增量或矛盾，不改写历史条目
- `## 关联` 用自然语言描述关系，不预设固定分类
- 所有链接使用 Obsidian wikilink 语法 `[[文件名]]`
