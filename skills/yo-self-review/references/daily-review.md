# 每日复盘模式

## 输入来源

用户可指定日志。

若无指定，读取：

`$NOTES_DIR/02-我的复盘/01-每日思考/{YYYY-MM}/{YYYY-MM-DD}.md`

重点读取：

* 今日要事
* 时间分配
* 今日收获
* 今日想法
* 今日任务
* 已有的今日洞察，如果存在

可以阅读文中提到的关联文件，但只在关联文件能帮助判断当天价值时才阅读。

## 分析目标

必须判断：

1. 今天是否推进了最重要主线？
2. 今天真正有价值的产出是什么？
3. 今天最大的低价值消耗是什么？
4. 今天暴露了什么判断偏差？
5. 是否有可沉淀资产？
6. 明天唯一最重要的聚焦点是什么？

## 今日判定标准

从以下四类中选择一个：

* 高质量推进：主线清晰，时间主要投入高价值任务，并形成明确成果。
* 基本合格：有一定成果，但存在明显效率问题。
* 偏离主线：低价值事务明显挤占主线任务。
* 严重偏离：大部分时间消耗在非核心事务，主线几乎没有推进。

## 输出位置

追加到当天日志结尾：

`## 今日洞察`

## 输出格式

```markdown
## 今日洞察

**今日判定**：高质量推进 / 基本合格 / 偏离主线 / 严重偏离。用一句话说明原因。

**关键产出**
- 只列真正产生价值的 1-3 件事。

**最大问题**
- 指出今天最影响效率或判断质量的问题。

**决策修正**
- 提炼 1 条以后可以复用的决策规则或行为约束。

**可沉淀资产**
- 只输出有价值的沉淀项。
- 可选类型：决策规则 / 经验教训 / Prompt / SOP / 文章素材。
- 没有明显沉淀时，写：暂无明显沉淀。

**明日唯一重点**
- 给出明天最重要的一件事，并说明为什么。
```

## 输出约束

* 总字数控制在 300-500 字。
* 不要复述当天做了什么。
* 只输出最关键的 3-5 个洞察。
* 准备、研究、梳理不算成果，除非形成可复用资产。
