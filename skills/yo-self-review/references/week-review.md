# 每周复盘模式

## 输入来源

用户可指定复盘文件。

若无指定，先确认今天日期（以周一为一周第一天），读取当前周复盘日志 `$WEEKLY_NOTE_PATH`（将 `{START}` / `{END}` 替换为本周一、本周日日期），并逐日读取本周每日复盘 `$DAILY_NOTE_PATH`（替换 `{YYYY}` / `{YYYY-MM}` / `{YYYY-MM-DD}`）。

## 分析目标

周复盘不是总结每天做了什么，而是识别一周的趋势。

必须判断：

1. 本周是否推进了长期主线？
2. 本周时间主要投向高价值任务还是低价值事务？
3. 本周是否出现重复问题？
4. 本周是否存在目标漂移、假忙、工具折腾、输入过量、输出不足？
5. 本周有哪些真正可沉淀资产？
6. 下周唯一主线是什么？
7. 下周必须禁止什么？

## 本周判定标准

从以下四类中选择一个：

* 高质量推进：主线清晰，时间主要投入高价值任务，并形成明确成果或资产。
* 基本合格：有成果，但存在时间分配或主线聚焦问题。
* 偏离主线：低价值事务、工具折腾、输入过量或目标漂移明显影响核心推进。
* 严重失焦：本周大部分时间消耗在非核心事务，长期目标几乎没有推进。

## 输出位置

追加到周复盘日志结尾：

`## 本周洞察`

## 输出格式

```markdown
## 本周洞察

**本周判定**：高质量推进 / 基本合格 / 偏离主线 / 严重失焦。用 1-2 句话说明原因。

**本周真正推进的事**
- 只列真正推动长期目标的 1-3 件事。

**本周最大问题**
- 指出本周最严重的问题，尤其是重复出现的问题或时间黑洞。

**趋势观察**
- 识别本周反复出现的行为模式，例如：工具折腾、假忙、研究替代输出、主线切换、输入过量、完美主义。

**决策修正**
- 提炼 1-3 条以后可以复用的决策规则或行为约束。

**可沉淀资产**
- 只输出有价值的沉淀项。
- 可选类型：决策规则 / 经验教训 / SOP / Prompt / 工作流 / 项目经验 / 文章素材。
- 没有明显沉淀时，写：暂无明显沉淀。

**下周唯一主线**
- 给出下周最重要的一个方向，必须单一、清晰、可执行。

**下周禁止事项**
- 明确下周必须避免的 1-3 件事。
```

## 输出约束

* 总字数控制在 500-800 字。
* 不要按日期复述。
* 不要复述每日任务列表。
* 优先分析趋势，而不是事件。
* 必须识别重复问题。
* 必须指出本周最大时间黑洞。
* 准备、研究、梳理不算成果，除非形成可复用资产。
