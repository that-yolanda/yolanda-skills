# yo-wiki-atom

将用户任意输入拆解为知识原子，写入原子库，如用户要求或判断为可复用资产时保存进资料库。不负责提炼规律、更新画像或写入知识库。

`$WIKI_DIR` 是系统环境变量，指向知识库根目录。下文所有路径通过 `$WIKI_DIR` 引用。首次使用若未配置，按 [references/first-time-setup.md](references/first-time-setup.md) 引导用户设置。

## User Input Tools

当本 skill 需要提示用户时，按此工具选择规则（优先级顺序）：

1. **优先使用当前 agent runtime 内置的用户输入工具** — 如 `AskUserQuestion`、`request_user_input`、`clarify`、`ask_user`、`questionnaire` 或等效工具。
2. **回退到纯文本**：若无此类工具，输出编号的纯文本消息，让用户回复选择的编号/答案。
3. **批量规则**：若工具支持单次调用多个问题，将所有适用问题合并为一次调用；若仅支持单问题，按优先级逐个提问。

下文对 `AskUserQuestion` 的具体提及均为示例 — 其他 runtime 按规则替换为本地等效工具。

## 可用工具

- 网页搜索与访问：`web_search` `web_fetch`
- 需登录/cookie 的网页内容读取：`opencli`
- 原子库读写：`yo wiki search` / `yo wiki add`（通过 `-h` 查看参数）

## 必须阅读

- `$WIKI_DIR/使用说明.md` — 跨层权限，禁止直接编辑 `atoms.jsonl`
- `$WIKI_DIR/原子库/使用说明.md` — atom 字段、type 定义、价值标准
- `$WIKI_DIR/资料库/使用说明.md` — 资料保存规范（如需保存资料时读取）

## 禁止行为

- 严禁直接查看或编辑 `atoms.jsonl`，只通过 `yo wiki add` 写入
- 严禁直接修改 `$WIKI_DIR/` 中的任何信息，除了通过 `yo wiki add` 的更新

## 做什么

- 信息读取：用户可能提供链接或文件，先通过可用工具获取原文内容，再从中提取。
- 信息理解：收到用户输入后，先归纳整体，识别信息是否有错误/歧义，如有则告知用户确认。
- 价值判断：判断是否值得记录——标准同原子库使用说明："下个月能不能用上"。用不上就不动，不为了记录而记录。
- 知识提炼：值得记录时，从输入中提取独立的知识点，按原子库的 type 定义归类，通过 `yo wiki add` 写入。一条输入可拆多条原子，每条一个独立知识点。不必拆太碎，只提取有长期价值的。
- 信息标注：`confidence` 标注知识点在原文中的可靠程度（原文中有明确证据的标 `high`，原文自己也不确定的标 `low`），不加 AI 自身的推断或观察。
- 资料保存：当用户提到需要保存资料，或当判断该资料为高价值是后续可复用的资产时，对资料整理，排版并保存进 `资料库` 目录，具体参考`$WIKI_DIR/资料库/使用说明.md` 规范进行整理
