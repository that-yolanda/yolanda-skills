# 知识库系统说明

本文档说明知识系统的架构规则和各层的分工边界。不说明各层内部的字段定义或格式规范——这些见各目录的 `使用说明.md`。

## 初始化（从模板创建你自己的知识库）

```bash
# 1. 复制本模板到你的知识库位置
cp -r vault-template ~/path/to/我的知识库

# 2. 设置环境变量 WIKI_DIR 指向它
#    （由 agent 经 first-time-setup 引导写入 $YO_CONFIG_HOME/config.env；
#     macOS/Linux 在 shell rc 中 source，Windows 用 setx；详见仓库 .env.example）

# 3. 开始使用：安装 yo-* 知识库 skill（见仓库 README），
#    通过 yo-opentalk 对话即可沉淀知识
```

> 本目录是**模板**，仅含框架与使用说明，不含任何个人数据。复制后在你自己的副本中填充内容。

## 核心原则

- 使用说明：使用说明是整个系统的最高规范，阅读/修改某个文件夹或文件之前，必须先阅读该目录的 `使用说明.md` 再操作
- 信息流向：原子库 → 知识库，知识库只能是从原子库提炼生成，不直接从对话写入
- 价值标准：价值判断是最高标准，判断依据就是 "下个月我还能不能用上"
