# 我的画像使用说明

本文档说明个人画像各文件的定位、收录规则和格式。画像内容是原子库在个人维度的派生层，部分内容从原子库提炼，部分在对话时直接记录。原子字段定义见 [原子库/使用说明.md](../原子库/使用说明.md)。

## 我的经历（过去）
- 定义：用户自身真实发生的行为、决定、实验结果。
- 记录：个人实验与结果、关键决策与过程、技能进步节点。例如：从零搭建了知识库系统；用某方法涨粉 1000。
- 不记录：纯外部知识（归原子库→知识包），心情波动，未产生结果的动作
- 来源：提炼时从原子库中 `type=case` 或 `anti-pattern`、内容描述用户自身行为的原子提取；对话时直接追加
- 格式：`- 描述事件 [date:: YYYY-MM-DD] [id:: atom_id] #tags`。如无对应原子可不写 `id`
- 规则：只追加，不修改已有条目

## 我的当前状态（现在）
- 定义：当前对话结束时的上下文快照，帮助下次对话接上。
- 记录：当前关注、最近动态、待处理、下次对话入口
- 不记录：长期目标（归我的目标），稳定偏好（归我的偏好），已解答的疑问（归我的疑问）
- 来源：每次对话结束时更新
- 格式：4 个小节，每次覆盖更新，不追加
    - `## 当前关注`：现在主要在做什么
    - `## 最近动态`：最近发生的值得跟踪的事
    - `## 待处理`：开了头还没收尾的
    - `## 下次对话入口`：如果回来说 X，应该走 Y

## 我的目标（未来）
- 定义：用户的短期和长期目标。
- 记录：想达成什么、为什么重要、时间窗口。例如：Q3 前完成知识库系统搭建并能日常使用
- 不记录：模糊愿望（"想变得更好"），纯任务清单
- 来源：用户手动设置；对话时发现新目标可提议追加
- 格式：`- [状态] 目标描述 — 为什么重要 — 预计时间窗口`
- 状态：待执行 `- [ ]`  /进行中 `- [/]` / 已完成`- [x]` / 放弃`- [-]`，完成或放弃后标注不删除

## 我的偏好（底色）
- 定义：可移植的个人偏好配置，每个章节可独立拷贝到 agent 系统提示词中使用。
- 记录：文档排版、文件管理、AI 回复风格、代码风格、工具选择、**音乐偏好**等任何跨 agent 通用的行为约定
- 不记录：仅适用于单一项目或单一 agent 的临时配置
- 来源：对话中用户表达偏好时直接记录
- 格式：按场景分 `##` 章节，每章内用无序列表。章节内容自包含，可直接拷贝使用。新增偏好追加到已有章节末尾或新建章节

> yo-utils-music 的音乐偏好读取本文件中的一个 `##` 章节（如 `## 音乐偏好`）。

## 我的疑问（盲区）
- 定义：用户尚未想清楚的开放问题。
- 记录：对方向、策略、方法等的不确定。例如：哪个平台值得重点投入；当前定价策略是否可持续
- 不记录：已解决的问题（标注后保留），纯知识性问题（适合直接搜索的）
- 来源：提炼时从原子库 `type=insight`、`confidence=speculation` 且为用户自身开放性问题的原子提取；对话时直接追加
- 格式：`- [状态] 疑问描述 [date:: YYYY-MM-DD]`，解答后换行追加 `  答案：xxx [date:: YYYY-MM-DD]`，并更新问题状态
- 状态：未解决`- [?]` / 解决 `- [x]` / 放弃取消`- [-]`
