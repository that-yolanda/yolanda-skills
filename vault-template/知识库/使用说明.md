# 知识库使用说明

本文档说明知识库的专题定义、文档格式和专题生命周期。原子字段定义见 [原子库/使用说明.md](../原子库/使用说明.md)，跨层权限见 [根目录 使用说明](../使用说明.md)。

> 下面的「现有专题」为**示例**，展示如何定义一个专题。请按自身领域增删调整。

## 现有专题（示例）

提炼原子时，读每个专题的 `判断标准` 和对应专题文档的 frontmatter `description`，按语义判定归属。如发现有价值但无匹配专题的原子，按[专题生命周期](#专题生命周期)流程提议新增，不可强行塞入已有专题。

- 商业判断
  - 定义：生意层面的判断原则、决策方法、商业案例。
  - 记录：定价策略、商业模式判断、收入结构设计、商业决策过程与结果。例如：引流款和利润款价格差 10 倍；先验证需求再开发产品。
  - 不记录：不直接涉及赚钱/生意的通用思维方法（归认知框架），纯内容运营技巧（归内容方法）。
  - 判断标准：是否直接影响商业结果？是否跟赚钱/生意决策直接相关？

- 内容方法
  - 定义：内容生产、分发、平台策略的方法和案例。
  - 记录：选题方法、文案结构、标题公式、平台规则与策略、内容与商业模式的匹配方式。
  - 不记录：不涉及内容生产分发的纯商业判断（归商业判断），纯 AI 提示词技巧。
  - 判断标准：是否直接服务于内容创作或分发？

- 认知框架
  - 定义：思维模式、认知方法和复盘框架。
  - 记录：元认知工具、决策框架、系统思维、复盘方法、反模式识别。
  - 不记录：直接涉及商业决策的思维方法（归商业判断）。
  - 判断标准：是否涉及通用思维、认知或复盘方法？

## 专题生命周期

### 新增专题

发现无匹配专题的原子时：

- 输出建议方案：专题名 + `description` + 可归入的原子示例 + 解决什么问题
- 用户确认/修改后，创建专题文件（含 frontmatter `description`），并在本文档现有专题列表中追加新条目
- 将原子提炼到新专题

### 废弃专题

- 专题下无内容且 6 个月内无新原子归入 → 可提议废弃
- 专题内容被其他专题完全覆盖 → 可提议合并
- 废弃后专题文件移入 `99-归档/`，本文档列表标注已废弃

## 文档格式

每个专题一个 Markdown 文件，命名 `{专题}.md`。

### Frontmatter

```yaml
---
description: 一句话说清本专题的覆盖范围和解决的问题
atom_count: 23
updated: 2026-06-02
---
```

### 条目格式

无序列表，dataview 行内语法标注元信息：

```markdown
- 提炼后的知识点陈述 [id:: atom_id] [date:: YYYY-MM-DD] #tag1 #tag2
  原文：片段或来源说明
```

### 小节标题

每个文档按原子 `type` 分节：

- `## 原则` ← `principle`
- `## 方法` ← `method`
- `## 工具` ← `tool`
- `## 洞察` ← `insight`
- `## 案例` ← `case`、`anti-pattern`

某小节无内容时可省略。新追加的条目放在对应小节的末尾。

### 示例

```markdown
---
description: 生意层面的判断原则与决策案例，覆盖定价、商业模式、收入结构等
atom_count: 1
updated: 2026-06-02
---

# 商业判断

## 原则

- 定价即产品，价格本身反向筛选用户群 [id:: 2026-05-015] [date:: 2026-05-15] #商业模式 #定价
  原文：引流款和利润款的价格差最好是 10 倍
```
